Инструмент "Сравнение производительности кода" предназначен для анализа производительности различных способов решения одной и той же задачи.
Основная идея инструмента заключается в возможности проверить, какой из методов решения задачи будет наиболее производительным.

Готовый тестовый пример доступен по команде Открыть пример:

Схема работы инструмента


  1. Выполнить код инициализации;
  2. Начать замер времени выполнения
  3. Выполнять блок кода, пока не истечет заданное время
  4. Закончить замер времени выполнения
  5. Проверить соответствие результата ожидаемому (об этом ниже)

Код инициализации

Это блок кода, выполняемый перед каждым блоком кода и предназначенный для инициализации каких-либо данных - передача общих параметров, получение данных из информационной базы и прочее.
Важное замечание - выполнение кода инициализации происходит до начала замера времени выполнения, а значит никак не влияет на показатели производительности тестируемых блоков кода.

Некоторые настройки выполнения

  • Где выполнять - позволяет выбрать среду выполнения - клиент или сервер;
  • Время выполнения - время, в течение которого будет выполняться каждый блок кода. Каждый тестируемый блок кода выполняется раз за разом до тех пор, пока не истечет это время.

Тестируемые блоки кода

Это список блоков, в каждом из которых находится свой вариант решения исходной задачи. Для каждого блока задается Представление, которое должно кратко и понятно описывать, какая именно реализация находится в этом коде.
После замера в данной таблице заполняется две колонки:
  • Количество итераций - сколько раз программа успела выполнить данный блок кода за время, указанное в поле Время выполнения;
  • Среднее время выполнения - сколько времени в среднем было затрачена на одну итерацию выполнения кода из данного блока.

Проверка корректности выполнения

Инструмент имеет удобный способ автоматической проверки правильного вычисления результата выполнения. Для этого:
  • В коде инициализации необходимо указать ожидаемый результат, записав его в переменную ОжидаемыйРезультат;
  • В каждом тестируемом блоке необходимо передавать результат вычисления в переменную Результат.
